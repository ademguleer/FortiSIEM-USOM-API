# FortiSIEM - USOM Threat Intelligence Integration (Decoupled Architecture)

A highly optimized, two-tier (decoupled) Python integration for FortiSIEM to ingest the USOM (National Cyber Incident Response Center of Turkey) Malicious IP threat feed. 

## 🚨 The Problem: Monolithic Plugin Limitations
FortiSIEM handles external Python plugins via its Java AppServer (`phoenix` daemon). When dealing with large, heavily paginated APIs like USOM (which returns over 14,500+ IOCs across 720+ pages with strict WAF rate limits), fetching the entire dataset takes approximately 6-8 minutes. 

Because the FortiSIEM AppServer has a hardcoded thread timeout limit (typically 5 minutes) to prevent memory hogging, long-running monolithic scripts are silently killed, resulting in:
* `java.io.IOException: Stream closed` errors in logs.
* `0 - 0` entries updated in the GUI.
* Missing or truncated Threat Intelligence data.

## 💡 The Solution: Decoupled Architecture
To bypass the AppServer thread limits without losing data, this project implements a **Decoupled Architecture** utilizing two lightweight scripts:

1. **The Downloader (`usom_downloader.py`):** A standalone Python script scheduled via Linux `cron`. It safely navigates the API pagination, respects WAF rate limits via sleep intervals, deduplicates identical IPs in memory to prevent FortiSIEM Unique Constraint Violations, and dumps a sanitized dataset to a local `.csv` file.
2. **The Feeder (`usom_threatfeed.py`):**
   A minimal FortiSIEM plugin script. Instead of making HTTP requests to the internet, it instantly reads the local `.csv` file generated by the Downloader and pushes it to the AppServer in fractions of a second.

**Result:** Zero timeouts, zero memory hogging, and 100% data ingestion reliability.
